# Adoption Signals

Lightweight tracking guide for early adoption of the bmad-ui project. Review weekly using the template below to detect traction signals and map them to backlog actions.

---

## Signal Inventory

| Signal | Where to find | Cadence |
|---|---|---|
| Stars | Repository main page / Insights → Traffic | Weekly |
| Forks | Repository main page / Insights → Traffic | Weekly |
| Issues opened / closed | Issues tab | Weekly |
| Discussions started | Discussions tab | Weekly |
| First-time contributors | Insights → Contributors | Weekly |
| Unique clones | Insights → Traffic → Git clones | Weekly |
| Unique views | Insights → Traffic | Weekly |

All signals are readable under **Insights → Traffic** and **Insights → Contributors** — no API tokens or automation required.

## Signal-to-Backlog Mapping

Use this table to translate signal patterns into concrete follow-up actions.

| Signal pattern | Likely implication | Suggested follow-up |
|---|---|---|
| Stars growing, forks low | Interest but adoption friction | Review quickstart and onboarding path |
| Issues spike with `bug` label | Reliability problem surfaced | Triage → Epic 4/8 items |
| First-time contributor PRs appear | Onboarding working | Ensure contribution guide stays current |
| Discussion questions repeat | Docs gap | Update relevant guide, consider FAQ section |
| Star growth stalls after initial spike | Positioning gap | Review README, project description, and topics |
| Clones high but stars low | Evaluation without commitment | Check that value proposition is clear in README |
